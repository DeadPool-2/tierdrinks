# TierDrinks ⚡

Рейтинг энергетиков и газировок на двоих (Миша/Тимур) с авто тир-листом.
Оценка по 2 осям (вкус, послевкусие), тир-листы S–F,
два профиля без пароля.

**Фичи:**
- **Каталог 200 SKU / 24 бренда** — энергетики (Monster, Red Bull, Burn, RU-бренды,
  Rockstar, Reign, Bang, NOS…) и газировки (Mountain Dew, Dr Pepper, Coca-Cola…).
  Все карточки с официальными студийными рендерами (98%), источники — в `sourceUrl`
  записей `src/seedExtra.json`.
- **Категории** — переключатель ⚡ Энергетики / 🥤 Газировка в каталоге и тир-листе.
- **Двухуровневый фильтр** бренд→линейка (Monster: Ultra/Juiced/Rehab/Java/Reserve;
  Red Bull: Classic/Editions) + поиск.
- **Лог дегустаций** — каждая оценка = выпитая банка с датой; попробовал снова →
  новая запись. История со спарклайном, «удалить последнюю».
- **Цена вручную + тренд** — датированные точки цены, мини-график.
- **Тир-листы** — общий / мои / друга, отдельно по категориям, плитки с оценкой,
  клик открывает карточку.
- **Раздел «Всё»** — выпито по брендам, 💰 кто сколько потратил (банка = цена на
  момент дегустации), прогресс каталога, любимые вкусы и теги, 🏆 топ напитков,
  распределение по тирам, динамика цен, совпадение вкусов и щедрость на баллы.
- **Бэкапы** — дневная ротация `data/backups/` (7 копий) на каждой записи +
  кнопка «Скачать бэкап (JSON)» внизу «Всё» (`GET /api/export`).

**Стек:** двойной zero-deps бэкенд — `server.js` (Node, dev) и `api.php`
(прод, PHP 5.6-совместимый порт), общий JSON-стор. Статичный SPA (vanilla JS).

## Локально

```bash
npm start           # http://localhost:3000
npm run dev         # то же с авто-перезапуском (node --watch)
npm test            # смоук-тесты бэкенда (node:test): merge/prune/tombstone/export
```

Данные — `data/db.json` (создаётся из сида при первом старте, гитигнорится).

## Каталог и картинки

- База: `src/seedData.js` (Monster, Red Bull, Burn) + `src/seedExtra.json`
  (ресёрч-линейки). Бренд с полной линейкой в seedExtra **вытесняет** старые
  записи из seedData. `src/imageMap.json` — id→картинка, бинарники в
  `data/images/` (в гите, force-add).
- После любых правок каталога: `node scripts/build-seed.mjs` → `data/seed.json`
  (его читает api.php).
- Сид мёржится в живую базу на каждом старте/запросе: новые SKU доезжают без
  потери оценок; удалённые напитки не воскресают (tombstone); сид-сироты без
  оценок вычищаются (prune); цена следует за сидом, пока её не правил юзер.
- Пайплайн официальных рендеров: workflow-агент на бренд (WebSearch/WebFetch по
  ретейлерам: Kroger UPC-паттерн, Amazon `_SL1500_`, РФ-магазины) →
  `node scripts/apply-lineup.mjs <lineups.json>` (линейки + фото) или
  `node scripts/apply-images.mjs <pairs.json>` (точечные замены {id,url}).
  Скачанное валидируется (magic-байты, ≥10KB). Тяжёлые файлы жать:
  `sips -Z 700` (+ png→jpg для непрозрачных гигантов).
- Свой напиток — вкладка «＋» в UI.

## Деплой на прод (Beget shared, PHP 5.6)

Прод: `http://mlepet70.beget.tech`. В веб-терминале Beget:

```bash
cd ~/tierdrinks && git pull && bash deploy-beget.sh
```

Скрипт кладёт статику+`api.php` в docroot, картинки в `/images`, свежий
`data/seed.json` — и **не трогает** живой `data/db.json` (оценки переживают
деплой, каталог домёрживается на лету). После деплоя поднять cache-bust
`?v=N` в `public/index.html`, если менялись js/css.

⚠️ `api.php` держать PHP 5.6-совместимым: `array()`, `function(){}`, без
`??`/стрелок/spread.

## Деплой на VPS (альтернатива, drinks.bondapp.ru)

```bash
./deploy.sh          # rsync + systemd + nginx (ssh alias "vps")
```

> ⚠️ С машины за прокси (fake-IP) SSH к VPS рвётся: выключить прокси или
> DIRECT-правило для IP/домена.
